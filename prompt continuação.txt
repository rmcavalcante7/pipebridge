====================================================================
🎯 PROJECT CONTEXT
==================

We are building a Python SDK for Pipefy with the following architecture:

* client/

  * httpClient.py → Responsible for GraphQL communication

* builders/

  * queryBuilder.py → Responsible for building GraphQL queries dynamically

* services/

  * cardService.py
  * phaseService.py
  * fileService.py
    → Contain all business logic and Pipefy operations

* models/

  * card.py
  * pagination.py
    → Strongly typed domain models

* exceptions/

  * exceptions.py
    → Custom exception handling (RequestError, InvalidParameterError, etc.)

---

The SDK follows a layered architecture:

LOW LEVEL (RAW API)
→ Methods returning Dict[str, Any]

HIGH LEVEL (MODEL)
→ Methods returning typed models (Card, PageInfo, etc.)

==============================

⚠️ ERROR HANDLING STANDARD
==========================

* NEVER suppress exceptions
* ALWAYS wrap errors with RequestError
* ALWAYS include:

* ALWAYS use:
  raise RequestError(...) from exc

====================================================================
🧠 MODEL USAGE RULES (CRITICAL)
===============================

DO NOT instantiate models like:

❌ Card(**data)

ALWAYS use:

✔ Card.fromDict(data)
✔ Card.fromResponse(response)
