## 🎯 CONTEXT

We are building a system that interacts with the Pipefy API (GraphQL).

### Known constraints:

* `card.fields` DOES NOT provide phase information
* Fields may belong to **multiple phases**
* Pipe structure must be fetched separately:

  * `pipe.phases.fields`
* Business rules MUST NOT rely on labels (only `field_id`)
* GraphQL mutations do NOT support conditional execution (no IF logic)

   * field_id
   * current phase
   * contextual data (user, status, etc.)
3. Is:

   * extensible
   * configurable (JSON / DB)
   * decoupled from API layer
   * production-ready (SDK quality)
